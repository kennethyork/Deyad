AI Providers

Deyad supports multiple AI backends. Ollama is the default and recommended provider for fully local inference.

Supported Providers

ProviderLocalAPI KeyNotes
Ollama✅NoDefault. Fully local. No data leaves your machine.
OpenAI❌YesGPT-4o, GPT-4.1, o3, etc.
Gemini❌YesGoogle Gemini models
OpenRouter❌YesAggregator — access many models via one API
xAI❌YesGrok models

Ollama (Default)

No configuration needed. Deyad connects to http://localhost:11434 by default.

Install Ollama

curl -fsSL https://ollama.com/install.sh | sh

Pull a Model

# Recommended default
ollama pull qwen3:8b

# Larger model for more capable reasoning
ollama pull qwen3.5:27b

# Code completion model (used for FIM)
ollama pull qwen2.5-coder:7b

Custom Ollama Host

# Connect to a remote Ollama instance
export OLLAMA_HOST=http://192.168.1.100:11434

OpenAI

export OPENAI_API_KEY=sk-...
deyad -m gpt-4o "your prompt"

Gemini

export GEMINI_API_KEY=...
deyad -m gemini-2.5-pro "your prompt"

OpenRouter

export OPENROUTER_API_KEY=sk-or-...
deyad -m openrouter/anthropic/claude-sonnet-4 "your prompt"

xAI

export XAI_API_KEY=xai-...
deyad -m grok-3 "your prompt"

Tips

  • Stay local: Ollama keeps all data on your machine — no API keys, no network calls, no telemetry.
  • Model size matters: Larger models (27B+) produce better reasoning and tool use, but require more VRAM.
  • Mix providers: Use Ollama for day-to-day work and a cloud provider for complex tasks.
  • FIM model: Set a separate fill-in-the-middle model for code completions in the Xed plugin or editor integrations.
