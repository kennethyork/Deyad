Automation & CI Guide

Use Deyad in scripts, pipelines, and GitHub Actions for automated code generation and maintenance.

Non-Interactive Mode

Use the -p (print) flag to run Deyad in scripts. It executes the prompt and prints the result to stdout — no interactive UI.

# One-shot task
deyad -p "update the README with the new feature description"

# Pipe output
deyad -p "summarize the changes in the last 5 commits" > summary.txt

Auto-Confirm Mode

Use -y to skip all confirmation prompts. The agent will automatically apply file changes and run commands.

deyad -y "fix all linting errors in src/"

Example: GitHub Action

Automate changelog updates after every push to main:

name: Automated Changelog

on:
  push:
    branches: [main]

jobs:
  update-changelog:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install Ollama
        run: curl -fsSL https://ollama.com/install.sh | sh

      - name: Pull model
        run: ollama pull qwen3:8b

      - name: Install Deyad CLI
        run: |
          git clone https://github.com/kennethyork/Deyad.git /tmp/deyad
          cd /tmp/deyad/deyad-cli/deyad-cli
          npm install && node build.mjs && npm link

      - name: Update changelog
        run: deyad -y -p "Update CHANGELOG.md based on the latest commits"

      - name: Commit changes
        run: |
          git config user.email "action@github.com"
          git config user.name "GitHub Action"
          git add CHANGELOG.md
          git commit -m "chore: automated changelog update" || echo "No changes"
          git push

Best Practices

  • Use scoped prompts — Instead of "refactor the whole repo", use "fix linting errors in src/utils.ts".
  • Sandbox your environment — Run in Docker or a disposable VM to prevent accidental destructive commands.
  • Check the output — Always review CI logs to verify the agent behaved as expected.
  • Pin your model — Use -m model:tag for reproducible results across runs.
