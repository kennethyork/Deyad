Global Configuration

Set your default model, preferences, and persistent agent instructions that apply across all projects.

Configuration Directory

Deyad stores global configuration in ~/.deyad/. This directory is created automatically on first run.

Configuration Files

FilePurpose
~/.deyad/config.jsonStructured settings — model, approval mode, etc.
~/.deyad/instructions.mdPersistent natural-language instructions for the agent
~/.deyad/audit.logTool execution audit log (auto-generated)
~/.deyad/sessions/Saved conversation sessions

config.json

Create or edit this file to set your defaults:

{
  "model": "qwen3.5:27b",
  "autoConfirm": false
}

Options

KeyTypeDescription
modelstringDefault Ollama model name
autoConfirmbooleanSkip confirmation prompts by default

instructions.md

This file lets you define persistent rules and preferences for the agent. It's included in every conversation's system prompt.

Example

# Global Agent Instructions

- Always use TypeScript for new files.
- When refactoring, ensure all existing tests still pass.
- Prefer functional components and hooks in React.
- Use npm, not yarn.
- Always explain complex regex in a comment.

Project-Level Overrides

You can override global settings per-project by placing a DEYAD.md (or .deyad.md) file in your project root. Project instructions take priority over global ones.

Example DEYAD.md

# Project Instructions

This is a React + Express + SQLite project.
- Backend is in src/server/, frontend in src/client/.
- Use Vitest for testing.
- All API routes should validate input with zod.

Command-Line Overrides

Flags always take precedence over config files:

# Override model for one command
deyad -m qwen3:8b "quick task"

# Override auto-confirm
deyad -y "apply all fixes"
