Interactive Mode Guide

The REPL is the heart of Deyad CLI — a continuous conversation with your AI agent.

Starting the REPL

Run deyad without arguments to enter interactive mode:

deyad

Key Features

  • Streaming Responses — Watch the agent's response as it's generated in real-time.
  • Tool Execution Visibility — See exactly which tools the agent invokes (file read, shell command, git, etc.).
  • Live Diffs — Review file changes before they're applied.
  • Session Persistence — Conversations are automatically saved and can be resumed.

Slash Commands

CommandDescription
/helpShow all commands
/model <name>Switch to a different Ollama model
/add <file>Add a file to the conversation context
/drop <file>Remove a file from context
/diffShow all uncommitted changes
/undoRevert the agent's last file changes
/run <cmd>Run a shell command directly
/compactShow token count and message stats
/clearReset the conversation history
/sandboxToggle sandbox mode (restricts writes)

Tips for Success

  • Be specific — Instead of "fix this", try "fix the error in src/utils.ts on line 45".
  • Use context — The agent has RAG access to your codebase. Refer to files by name.
  • Iterate — If the agent makes a mistake, tell it: "That's not right, try [alternative approach]".
  • Use /add — Explicitly add important files so the agent has them in context.
