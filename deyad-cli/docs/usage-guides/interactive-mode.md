# Interactive Mode Guide

The Interactive Mode (REPL) is the heart of `deyad-cli`. It provides a continuous conversation with the agent, allowing you to iterate on code changes, ask questions, and observe the agent's thought process in real-time.

## Starting the REPL

Simply run the command without any arguments:

```shell
deyad-cli
```

## The Interface

When the REPL starts, you will see a prompt waiting for your input. The interface uses `readline` to provide a familiar terminal experience.

### Key Features

- **Streaming Responses**: Watch the agent's response as it is being generated by the LLM.
- **Tool Execution Visibility**: See exactly which tools (file read, shell command, etc.) the agent is invoking.
- **Live Diffs**: When the agent proposes a file change, you can review the diff before approving.
- **Auto-completion**: Use the `Tab` key to complete commands or file paths (if supported by your shell integration).

### Interaction Modes

You can switch between different levels of autonomy within the session:

1. **Suggest Mode (Default)**: The agent reads files and proposes changes. You must approve every file write and every shell command.
2. **Auto Edit Mode**: The agent can automatically apply patches to files, but still requires approval for shell commands.
3. **Full Auto Mode**: The agent can execute shell commands and write files without asking, provided it's within the sandbox.

To change the mode, you can use the `--approval-mode` flag when starting the CLI, or (if implemented) a command within the REPL.

## Keyboard Shortcuts & Controls

| Shortcut | Action |
| --- | --- |
| `Ctrl+C` (once) | Interrupt the current agent turn/tool execution. |
| `Ctrl+C` (twice) | Exit the REPL. |
| `Ctrl+L` | Clear the terminal screen. |
| `Up/Down Arrows` | Navigate through your command history. |

## Tips for Success

- **Be Specific**: Instead of "fix this", try "fix the error in `src/utils.ts` on line 45".
- **Use Context**: The agent has access to your codebase via RAG. You can refer to files by name.
- **Iterate**: If the agent makes a mistake, don't start over. Just tell it: "Actually, that's not right, please try [alternative approach]".
